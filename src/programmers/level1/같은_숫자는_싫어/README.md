# 같은 숫자는 싫어

## 문제 요약
배열 `arr`가 주어질 때  
연속해서 나타나는 같은 숫자는 하나만 남기고 제거하는 문제이다.

단, 숫자의 순서는 유지해야 한다.

예시  
`[1,1,3,3,0,1,1]` → `[1,3,0,1]`

---

## 풀이 아이디어
연속된 값만 제거하면 되기 때문에  
마지막으로 저장된 값과 현재 값을 비교하면 된다.

1. 결과를 저장할 `List`를 만든다.
2. 배열의 첫 번째 값은 무조건 결과에 추가한다.
3. 배열을 순회하면서
    - 현재 값이 마지막으로 저장된 값과 다르면 `List`에 추가한다.
4. 마지막에 `List`를 배열로 변환하여 반환한다.

---

## 핵심 로직
1. 결과를 저장할 `List` 생성
2. 배열의 첫 번째 값을 `List`에 추가
3. 배열을 순회하며
    - 마지막 저장 값과 현재 값을 비교
4. 값이 다르면
    - `List`에 추가
5. `List`를 배열로 변환하여 반환

---

## 복잡도

시간 복잡도 : O(N)

배열을 한 번 순회하면서 비교만 수행하기 때문이다.

---

## 개선 포인트
현재 풀이에서는 `listId`로 마지막 인덱스를 따로 관리하고 있다.

하지만 `list.size() - 1`을 사용하면 별도의 인덱스 변수를 두지 않아도 된다.

또한 `List` 대신 결과 배열을 바로 생성하는 방식으로도 구현할 수 있지만  
크기를 미리 알 수 없기 때문에 `List`를 사용하는 방식이 구현이 더 단순하다.
