# 모의고사

## 문제 요약
3명의 수포자가 문제를 찍는 방식이 각각 정해져 있다.  
정답이 담긴 배열 `answers`가 주어질 때, 가장 많은 문제를 맞힌 사람의 번호를 배열로 반환하는 문제이다.

각 수포자의 찍기 패턴은 다음과 같다.

1번 수포자: `[1,2,3,4,5]` 반복  
2번 수포자: `[2,1,2,3,2,4,2,5]` 반복  
3번 수포자: `[3,3,1,1,2,2,4,4,5,5]` 반복

정답 배열과 각 수포자의 패턴을 비교하여 맞힌 개수를 계산하고, 가장 높은 점수를 받은 사람을 반환한다.

예시  
`[1,2,3,4,5]` → `[1]`

---

## 풀이 아이디어
각 수포자의 정답 패턴은 일정한 길이를 가지고 반복되므로, 문제 번호를 패턴 길이로 나눈 나머지를 이용하면 현재 문제에서 찍는 답을 구할 수 있다.

정답 배열을 순회하면서 각 수포자의 패턴과 비교하여 맞은 문제 수를 기록한다.  
이후 세 사람의 점수 중 최대값을 구하고, 해당 점수와 같은 사람의 번호를 결과 리스트에 저장한다.

마지막으로 리스트에 저장된 값을 배열로 변환하여 반환한다.

---

## 핵심 로직
1. 각 수포자의 찍기 패턴 배열 `p1`, `p2`, `p3` 생성
2. 세 사람의 점수를 저장할 배열 `score` 생성
3. `answers` 배열을 순회하면서
    - `i % 패턴길이`를 이용해 현재 문제에서의 패턴 값 계산
    - 정답과 같으면 해당 사람의 점수 증가
4. 세 사람의 점수 중 최대값 `max` 계산
5. 점수가 `max`와 같은 사람의 번호를 리스트에 저장
6. 리스트를 배열로 변환하여 반환

---

## 복잡도

시간 복잡도 : O(N)

---

## 개선 포인트
현재 코드에서는 패턴 길이를 직접 숫자로 사용하고 있다.

예를 들어 `i % 5`, `i % 8`, `i % 10`과 같이 작성했는데, 
`p1.length`, `p2.length`, `p3.length`를 사용하면 패턴 길이가 변경되더라도 코드를 수정할 필요가 없어 유지보수 측면에서 더 안전한 코드가 된다.

또한 점수 계산과 최대값 비교 로직을 별도의 메서드로 분리하면 코드의 가독성을 더 높일 수 있다.
