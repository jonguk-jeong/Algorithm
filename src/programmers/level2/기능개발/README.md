# 기능개발

## 문제 요약
각 기능의 현재 진행도(progresses)와 하루 작업 속도(speeds)가 주어진다.

기능은 앞의 작업이 완료되어야 배포할 수 있으며, 뒤의 기능이 먼저 완료되더라도 앞 기능이 완료될 때 함께 배포된다.

각 배포마다 몇 개의 기능이 배포되는지 구하는 문제이다.

---

## 풀이 아이디어
각 기능이 완료되는 날짜를 먼저 계산한 뒤,  같은 배포 그룹을 큐로 관리한다.

- 기능 완료일 계산
```
(100 - progress + speed - 1) / speed
```

- 새로운 기능의 완료일이 현재 배포 기준보다 늦으면    지금까지 모인 기능들을 하나의 배포로 처리한다.

---

## 핵심 로직
1. 기능 완료일까지 걸리는 날짜 계산
2. 큐에 현재 배포 그룹의 완료일 저장
3. 새로운 기능의 완료일이 큐의 첫 번째 값보다 크면
    - 큐 크기를 결과 리스트에 저장
    - 큐 초기화
4. 반복 종료 후 마지막 큐 크기 추가

---

## 복잡도

시간 복잡도 : O(N)


공간 복잡도 : 
O(N)

## 개선 포인트
큐 없이 기준 완료일을 변수로 관리하는 방식으로도 해결할 수 있다.
