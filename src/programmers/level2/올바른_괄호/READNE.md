# 올바른 괄호

## 문제 요약
문자열 `s`가 주어질 때  
괄호가 올바르게 짝지어져 있는지 판단하는 문제이다.

올바른 괄호의 조건은 다음과 같다.

- `(` 는 반드시 이후에 `)` 로 닫혀야 한다.
- 닫는 괄호 `)` 가 먼저 나오면 올바르지 않다.
- 모든 괄호가 정확히 짝을 이루어야 한다.

예시

`"()()"` → `true`  
`"(())()"` → `true`  
`")()("` → `false`  
`"(()("` → `false`

---

## 풀이 아이디어
괄호의 짝을 맞추는 문제이기 때문에 **Stack 자료구조**를 사용한다.

1. `(` 가 나오면 스택에 넣는다.
2. `)` 가 나오면
    - 스택이 비어있다면 짝이 없으므로 `false`
    - 비어있지 않다면 하나 꺼낸다.
3. 문자열을 모두 확인한 뒤
    - 스택이 비어있다면 모든 괄호가 짝을 이룬 것이므로 `true`
    - 남아있다면 `false`

---

## 핵심 로직
1. 괄호를 저장할 `Stack` 역할의 `ArrayDeque` 생성
2. 문자열을 앞에서부터 순회
3. 현재 문자가 `(` 이면
    - 스택에 추가
4. 현재 문자가 `)` 이면
    - 스택이 비어있다면 `false` 반환
    - 아니라면 스택에서 하나 제거
5. 순회가 끝난 뒤
    - 스택이 비어있는지 확인하여 결과 반환

---

## 복잡도

시간 복잡도 : O(N)

---

## 개선 포인트

현재 풀이에서는 `ArrayDeque`를 사용하여 스택처럼 구현했다.

하지만 이 문제는 스택 대신 카운트 변수 하나로도 해결할 수 있다.

- `(` 이면 `count++`
- `)` 이면 `count--`
- 중간에 `count < 0` 이 되면 바로 `false`

이 방식은 메모리 사용이 더 적고 코드가 더 간결해진다.
