# 주식가격

## 🔍 문제 요약
초 단위로 기록된 주식가격이 담긴 배열 `prices`가 주어질 때,  
각 시점의 가격이 **몇 초 동안 떨어지지 않는지**를 구하는 문제이다.

가격이 떨어지는 순간까지의 시간을 계산하며,  
끝까지 가격이 떨어지지 않는 경우 **마지막 시점까지의 시간**을 계산한다.

---

## 🧠 풀이 아이디어
가격이 떨어지는 시점을 빠르게 찾기 위해 **스택(Stack)** 을 사용한다.

- 스택에는 **가격의 인덱스**를 저장한다.
- 현재 가격이 스택 top 인덱스의 가격보다 작아지면 **가격이 떨어진 것**이다.
- 이때 스택에서 pop 하면서 **가격이 떨어진 시점까지의 시간**을 계산한다.
- 순회가 끝난 뒤에도 스택에 남아있는 인덱스들은 **끝까지 가격이 떨어지지 않은 경우**이므로 마지막 시점 기준으로 시간을 계산한다.

---

## 🔎 핵심 로직
1. 스택에 **가격의 인덱스**를 저장한다.
2. 배열을 순회하면서
    - 현재 가격이 스택 top의 가격보다 작다면
        - 스택에서 pop
        - `현재 인덱스 - pop한 인덱스` 로 시간 계산
3. 순회가 끝난 후
    - 스택에 남아있는 인덱스는 끝까지 가격이 떨어지지 않은 경우
    - `전체 길이 - 1 - 인덱스` 로 시간 계산

---

## ⏱️ 복잡도 분석
- 시간 복잡도: **O(N)**  
  (각 인덱스는 스택에 최대 한 번 push / pop 된다)

- 공간 복잡도: **O(N)**  
  (스택에 인덱스를 저장)

---

## ⚠️ 개선 포인트
- 처음 풀이에서는 `if`로 pop을 처리했지만  
  **여러 가격이 동시에 떨어질 수 있으므로 `while`을 사용해야 한다.**
- 스택에는 **가격이 아니라 인덱스를 저장해야 시간 계산이 가능하다.**
- 이 문제는 이중 반복문으로도 해결할 수 있다. 
  다만 이 경우 시간 복잡도가 `O(N²)`이 되므로, 스택을 활용한 `O(N)` 풀이가 더 효율적이다.
