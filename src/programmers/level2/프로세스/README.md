# 프로세스

## 문제 요약
프린터에는 여러 문서가 대기하고 있으며 각 문서는 중요도(priority)를 가진다.

프린터는 다음 규칙으로 문서를 출력한다.

1. 큐의 가장 앞 문서를 꺼낸다.
2. 뒤에 있는 문서 중 더 높은 중요도가 있다면 현재 문서를 다시 큐의 뒤로 보낸다.
3. 그렇지 않으면 해당 문서를 출력한다.

특정 위치(location)에 있는 문서가 몇 번째로 출력되는지 구하는 문제이다.

---

## 풀이 아이디어
큐에는 문서의 인덱스를 저장하고, 실제 중요도는 `priorities` 배열에서 참조한다.

문서를 하나 꺼낸 뒤,  
큐에 남아있는 문서들을 한 바퀴 순회하며 더 높은 priority가 존재하는지 검사한다.

- 더 높은 priority가 존재하면 → 현재 문서를 큐 뒤로 이동
- 없다면 → 해당 문서를 출력 처리

출력 순서를 `order`로 관리하고, 마지막에 `location` 위치의 출력 순서를 반환한다.

---

## 핵심 로직
1. 문서의 인덱스를 큐에 저장
2. 큐에서 문서를 하나 꺼냄 (`current`)
3. 남은 큐를 한 바퀴 순회하며 더 높은 priority 존재 여부 검사
4. 더 높은 값이 있으면
    - `current`를 큐 뒤로 다시 넣음
5. 없다면
    - 해당 문서를 출력 처리 (`order++`)
6. 반복이 끝난 후 `location`의 출력 순서를 반환

---

## 복잡도

시간 복잡도 : O(N²)

---

## 개선 포인트
출력 순서를 기록하는 배열 없이도 해결할 수 있다.

문서를 출력할 때 `current == location`인지 확인하면 바로 결과를 반환할 수 있어  
불필요한 배열을 제거하고 코드 길이를 줄일 수 있다.

우선순위 큐를 사용하면 좀 더 쉬울 것 같다.
